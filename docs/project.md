# JIMAKUWOプロジェクトの流れ

## 【第1走者】Whisper（耳）
役割: 音声を聴いて、文字に起こす。

処理:
動画の音声を聴き取り、1文字・1単語単位で「何秒から何秒まで発音されたか」という細かいタイムスタンプ情報（words データ）をすべて記録します。

特徴:
耳は非常に良いですが、日本語の文脈を深く考えるのは苦手なため、「がんばって」を「岩盤って」と聞き間違えたり、誤変換（空耳）した漢字のまま出力してしまうことがあります。


## 【第2走者】校正AI（頭脳）
役割: 文章をまとめて読み、正しい日本語にリライト（校正）する。

処理:
Whisperが出した文章を、細切れにせず「全体の文脈」としてまとめて読み込みます。前後の繋がりから「あ、この『岩盤って音声音付き』は、本当は『がんばって音声認識』と言いたかったんだな」と見抜き、正しい漢字と表現に一括修正します。

特徴:
文脈を読む天才ですが、文字の形だけを修正するため、この時点では「新しく直した文字が、何秒に発音されたか」という時間のデータを持っていません。


## 【第3走者】アルゴリズム（パズル・修復）
役割: 校正前と校正後の文字を見比べて、時間を完璧に紐付け直す。

処理:
動的計画法（レーベンシュタイン距離など）を使い、Whisperが持っていた「間違っているけれど時間が正確なデータ」と、校正AIが作った「正しいけれど時間がないテキスト」を、1文字ずつパズルのようにガチガチに噛み合わせます。

特徴:
言葉の意味は1ミリも理解しませんが、数学的な計算によって、直した後の正しい文字に対して「1ミリもズレのない完璧なタイムスタンプ（発言時間）」を復元・付与します。


## 【第4走者】BudouX（デザイナー）
役割: 読みやすい位置で改行し、字幕ファイルを生成する。

処理:
Google製の BudouX を使い、「今日は ／ 音声認識の ／ テストを」のように、人間がパッと見て脳内で理解しやすい「美しい文節の区切り」を計算します。
その区切りをベースに、1行が長くなりすぎないよう（10〜20文字程度）にテロップを改行・分割し、最終的な字幕の規格である SRT形式 のファイルを生成します。


## SRT出力（成果物）
役割: 字幕ファイルを書き出す。

処理:
以上の流れで生成された字幕ファイルを書き出します。

## DeBERTaの連携をどうするか？
穴埋めが強いらしい。
Whisperで確率的に高いと思う言葉は優先度を下げて、わからなかった部分のみを考えさせるMASKという特殊なトークンに置き換えて出力させることができるらしい


jimaku-scripter/
├── main.py                 # 全体のエントリーポイント（起動用スクリプト）
└── src/
    # ─── 【基盤・共通設定ファイル群】 ───
    ├── config.py           # プロジェクトの各種設定（閾値など）を保持
    ├── exceptions.py       # システム固有の独自エラー・例外を定義
    ├── utils.py            # 汎用的な便利関数（時間の変換処理など）を格納
    │
    # ─── 【メインパイプライン】 ───
    ├── pipeline.py         # 全体のデータフローをコントロールする司令塔
    │
    ├── processor.py        # 1. 音声前処理：FFmpegで動画/音声を 16kHz WAV化
    ├── transcriber.py      # 2. 音声認識：Whisperで文字起こし ＋ タイムスタンプ、確信度（重み）の抽出
    │
    ├── refiner.py          # 3. LLMによる校正: プロンプトから校正テキストを得る
    ├── proofreader.py      # 4. テキスト校正：確信度の低い単語を DeBERTa で MASK穴埋め修正
    ├── aligner.py          # 5. 動的計画法（DP）で、修正後テキストとwhisperタイムスタンプを照合
    ├── formatter.py        # 6. 文節整形：BudouX を使い、人間が読みやすい位置で綺麗に改行
    │
    └── srt_writer.py       # 7. ファイル出力：確定したテキストと時間を SRT 形式で書き出し



指定なし : Whisper➡フィラー除去➡校正（DeBERTa➡DP）➡BudouX➡SRT（LLMをしない）
--wsp   : Whisper➡フィラー除去➡BudouX➡SRT （LLMも校正もしない）
--llm   : Whisper➡フィラー除去➡LLM➡BudouX➡SRT（校正（DeBERTa/DP）をしない）
--all   : Whisper➡フィラー除去➡LLM➡校正（DeBERTa➡DP）➡BudouX➡SRT（全工程する）
これらは今後処理のパターンが増える可能性があるので拡張性のある構築にしたい。

※ whisper処理ではすべての生データを保持する。出力もする。特に重み値、タイムスタンプを全てのセグメントで保持することが重要➡後で使うため）今後何かに使う可能性があるので全生データを保持し続けること。
※LLM処理後にもそのデータを出力するのでデータを保持し続けること。
校正（DeBERTa、DP）のそれぞれの処理後にその生データを出力する。また生データを保持し続ける。
※出力はすべてoutput/フォルダで行う。

複雑な処理は行わない。複雑になる場合は別の関数に切り出してわかりやすくする。すべての処理に関してプログラミング初心者でもわかるようにコメントアウトで説明をする。説明は日本語で、長すぎずわかりやすく書く。
可能な限り元々のソースコードを変更せず、機能を追加していきたい。
変更すべき点は、修正前後の比較を教えてほしい。

DPとは動的計画法（レーベンシュタイン距離）である。この処理でwhisperが保持する生データにあるタイムスタンプとDeBERTa校正後の出力のタイムスタンプを比較して元々の音声の声のタイミングと合うように字幕を作成する。

DeBERTa校正時にもタイムスタンプを保持し続ける。
重み値の比較は今後実装予定。
